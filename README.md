# STQ — Storage Room Quiz

A PWA quiz app for Lineville Elementary storage rooms, with three difficulty levels, multiple-choice / true-false / free-form questions, two-part questions, surprise bonus questions, Quick Mode timed quizzes, and seven leaderboards. Wrong-answer choices are generated by Claude on the fly with difficulty-aware closeness rules.

## Stack

- **Hosting + functions:** Cloudflare Pages (free, GitHub-integrated)
- **Database:** Cloudflare D1 (SQLite, free)
- **AI:** Anthropic Claude API (called server-side from a Pages Function)
- **Frontend:** Plain HTML/CSS/JS, no build step. Installs as a PWA.

## One-time setup

You'll need a Cloudflare account (free) and an Anthropic API key. The whole thing should take 15-30 minutes the first time.

### 1. Push to GitHub

Create a new GitHub repository (e.g. `stq`) and push this folder to it.

```bash
cd stq
git init
git add .
git commit -m "Initial STQ"
git branch -M main
git remote add origin https://github.com/<your-user>/stq.git
git push -u origin main
```

### 2. Install Wrangler (Cloudflare's CLI)

You'll use Wrangler once to create the database and apply the schema.

```bash
npm install
npx wrangler login
```

### 3. Create the D1 database

```bash
npx wrangler d1 create stq-db
```

It will print something like:

```
[[d1_databases]]
binding = "DB"
database_name = "stq-db"
database_id = "abcd1234-...-..."
```

Copy the `database_id` and paste it into `wrangler.toml`, replacing `<YOUR-DATABASE-ID>`.

### 4. Apply the schema

```bash
npm run db:apply:remote
```

### 5. Create the Cloudflare Pages project

In the Cloudflare dashboard:

1. **Workers & Pages → Create application → Pages → Connect to Git.**
2. Pick your `stq` repo.
3. **Build settings**:
   - Framework preset: `None`
   - Build command: *(leave empty)*
   - Build output directory: `public`
4. Click **Save and Deploy**. The first deploy will succeed but the app won't work yet — we need to bind the database and set env vars.

### 6. Bind the database to the Pages project

After the first deploy:

1. Open the Pages project → **Settings → Functions → D1 database bindings**
2. Add a binding: **Variable name** `DB`, **D1 database** `stq-db`. Add for both **Production** and **Preview**.

### 7. Set environment variables

In **Settings → Environment variables** (for both Production and Preview):

| Variable | Value |
|---|---|
| `ADMIN_PASSWORD` | a password you choose (you'll use this to sign into /admin) |
| `CLAUDE_API_KEY` | your Anthropic API key (`sk-ant-...`) |
| `SESSION_SECRET` | any long random string (e.g. `openssl rand -hex 32`) |
| `CLAUDE_MODEL` | (optional) defaults to `claude-haiku-4-5-20251001`. Use `claude-sonnet-4-6` for smarter (but pricier) distractors. |

Mark `ADMIN_PASSWORD`, `CLAUDE_API_KEY`, and `SESSION_SECRET` as **encrypted**.

### 8. Trigger a fresh deploy

Push any commit, or hit **Retry deployment** in the dashboard. The site will redeploy with the bindings in place.

You're done. Visit your `https://<your-project>.pages.dev/` URL.

## Adding questions

1. Visit `/admin` on your live site
2. Sign in with the `ADMIN_PASSWORD` you set
3. Add at least **20 regular questions per difficulty level** before that level can be played (the home page tells you how many more you need)
4. Mark whichever questions you want as "surprise bonus" — they'll appear randomly on top of the 20 in any quiz of that difficulty, and they're worth double points

### Question types

- **Multiple choice** — write the correct answer; the AI generates 3 wrong answers each time the quiz runs. Closeness depends on difficulty:
  - **Hard:** numeric answers within ±2; text answers very similar in form
  - **Medium:** numeric within ±3; text moderately related
  - **Easy:** numeric within ±10; text clearly different
- **True / False** — only "True" and "False" appear as choices
- **Free-form** — short typed answer. Graded with case- and punctuation-insensitive match against the correct answer plus any alternate spellings you supply
- **Two-part questions** — one prompt with Part A and Part B. Each part has its own type and answer, and each part scores independently

### Numeric flag

For multiple-choice numeric answers, check the "Answer is numeric" box. This tells the AI to keep wrong answers within the difficulty-specific ± range.

## Quick Mode

Players can toggle Quick Mode on the home page before starting. It adds a countdown timer:

- **Easy:** 20 minutes
- **Medium:** 10 minutes
- **Hard:** 5 minutes

The elapsed time is also tracked for leaderboard tiebreaks. To make it onto a Quick Mode leaderboard, the player must score 90% or higher.

---

## Leaderboards

Seven boards, all top 10:

1. **All-Time** — every attempt, every level
2. **Easy / Medium / Hard** — per-level boards
3. **Quick · Easy / Medium / Hard** — Quick Mode boards (90%+ to qualify)

Ranking is by percentage descending, then elapsed time ascending (faster breaks ties), then completion time.

## Install as an iPad app

After the site is live:

1. Open the URL in Safari on the iPad
2. Tap the Share button → **Add to Home Screen**
3. Confirm — STQ now has its own icon and runs full-screen

The PWA caches the page shell, so it launches instantly. Quiz content still needs network (Claude has to generate the distractors).

## Local development

```bash
npm install
npm run db:apply:local        # applies schema to a local D1 sandbox
echo 'ADMIN_PASSWORD = "dev"\nCLAUDE_API_KEY = "sk-ant-..."\nSESSION_SECRET = "dev"' > .dev.vars
npm run dev                   # http://localhost:8788
```

## Costs

- **Cloudflare Pages / D1 / Functions:** free for this scale (D1 free tier is 5 GB and 5 M reads/day; Pages Functions free tier is 100 K requests/day)
- **Claude API:** each quiz makes 1 API call per multiple-choice part — roughly 20 calls per quiz attempt. With Claude Haiku 4.5 that's a small fraction of a cent per quiz. A $5 credit goes a very long way.

## Resetting things

- **Wipe all scores:** `npx wrangler d1 execute stq-db --remote --command="DELETE FROM attempts;"`
- **Wipe all questions:** `npx wrangler d1 execute stq-db --remote --command="DELETE FROM questions;"`
- **Re-apply schema (additive — won't drop tables):** `npm run db:apply:remote`
